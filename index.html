1. System Overview

TatataPay is a fully-managed payment gateway system designed for iGaming operators and PSPs. This guide covers the setup and operation of Bangladesh payment channels — including bKash, Nagad, and Rocket.

Key Roles

RoleDescriptionResponsibility
Payment Company (You)System operatorManage the platform, process orders, monitor funds
MerchantYour client (gaming platform, etc.)Sends collection/payout orders via API
Code Merchant (供卡方)Account providerProvides bKash/Nagad accounts, monitors transactions
If you operate your own bKash/Nagad accounts (no third-party code merchant), you can skip the "Code Merchant" setup and manage cards directly.

3. Adding a bKash / Nagad Collection Card

To start receiving payments, you need to add your bKash or Nagad merchant account to the system.

1 Navigate to Resource Management → Add Card
2 Fill in the card details:
  • Code Merchant: Select the code merchant (leave blank if you manage your own accounts)
  • Channel Type: Select bKash or Nagad
  • Mode: Keep default (embedded identification code)
  • Account Number: Enter the bKash/Nagad phone number (e.g., 01XXXXXXXXX)
  • Remarks: Optional — add a label for easy identification
3 Save — the system will auto-generate:
  • A Monitoring Account (username)
  • A Monitoring Password

Keep these credentials — you'll need them for OTP monitoring setup.

4 Configure Risk Controls (see Risk Control section)
Multiple cards can share the same monitoring account. You can also modify the monitoring account independently.

4. Setting Up OTP Monitoring

OTP monitoring captures SMS verification codes from the bKash/Nagad account's phone and automatically forwards them to the system backend. This is essential for automated transaction verification.

Prerequisites

  • The phone that receives bKash/Nagad OTP SMS
  • Our monitoring app installed on that phone
  • Monitoring account credentials (generated when you added the card)

Setup Steps

1 Install the monitoring app on the phone linked to the bKash/Nagad account
2 Grant all required permissions — check that all permission indicators are green (top-left of the app). If not, tap "Get Permissions" (top-right).
3 Enter login credentials in the app:
Monitoring account: [your monitoring username] Login password: [your monitoring password] Server configuration: https://your-domain.com

Note: Include https:// prefix. Do NOT add a trailing slash /

4 Tap Login — the app will start capturing OTP messages automatically
Once connected, any bKash/Nagad OTP received on this phone will be automatically forwarded to the system backend for transaction verification.

Template to Send to Local Operator

Please install the monitoring app and log in with these credentials: Monitoring account: XXXXXXXX Login password: XXXXXXXX Server configuration: https://your-domain.com After logging in, make sure all permission indicators are green. The app will automatically capture and upload OTP codes.

5. Setting Up Transaction Monitoring

Transaction monitoring automatically captures bKash/Nagad transaction records and uploads them to the backend for automatic order matching.

How It Works

  1. A customer makes a payment to your bKash/Nagad account
  2. The monitoring system detects the incoming transaction
  3. Transaction details are uploaded to the backend
  4. The system automatically matches the transaction to the corresponding order
  5. If matched → order marked as Success, callback sent to merchant
  6. If not matched → flagged for Manual Review

Matching Methods

MethodDescriptionRecommended
Amount MatchingMatch by payment amount⚠️ Basic
Identification Code + AmountUnique code embedded in payment + amount✅ Recommended
Payment Info + AmountCustomer-submitted info + amount⚠️ Backup

6. Merchant Onboarding

1 Create Merchant Account: Go to Merchant Management → Add Merchant → Enter nickname, username, password → Submit
2 Copy Integration Credentials: Click the "Integration Info" copy button → Send to merchant's technical team
3 Configure Merchant Settings:
  • Fee Rate: e.g., 0.02 = 2% per transaction
  • Fixed Fee: Per-transaction fixed fee (mainly for payouts)
  • IP Whitelist: Add merchant's server IP for security (format: 182.234.96.152,47.232.188.71)
  • Account Allocation: Assign specific code merchants to this merchant
  • Bot Group ID: For Telegram accounting bot integration
4 Merchant integrates via API using the provided documentation. Our team assists with any technical issues during integration.

7. Payment Product Configuration

Configure different products with different fee rates for each merchant.

Path: System Management → User Product Management → Add

  • Merchant: Select target merchant
  • Product: Select the payment product (e.g., bKash Collection, Nagad Payout)
  • Fee Rate: Percentage fee — e.g., 0.02 for 2%
  • Fixed Fee: Per-transaction fee — e.g., 5 for 5 BDT per transaction

8. Collection Order Flow

Complete Flow

  1. Customer places an order on the merchant's platform
  2. Merchant sends a collection request via API
  3. System validates: API key, IP whitelist, merchant ID, channel ID
  4. System allocates an available bKash/Nagad account
  5. System checks account availability: enabled? balance OK? within limits?
  6. System generates a cashier page link
  7. Customer pays via the cashier page (bKash/Nagad)
  8. Monitoring detects incoming payment
  9. System matches transaction to order
  10. System sends callback to merchant → Order complete

Order Statuses

StatusMeaning
CreatedOrder created, not yet processed
AssignedAllocated to a specific bKash/Nagad account
ProcessingWaiting for customer payment
SuccessPayment confirmed
FailedPayment verification failed
TimeoutNo payment received within time limit
ClosedOrder cancelled

9. Payout (Disbursement) Flow

Payouts allow merchants to send money from the system to customer bKash/Nagad accounts.

Methods

  • Single Payout: One-by-one disbursement via API
  • Bulk Payout: Batch processing multiple payouts at once
  • Manual Payout: Process from admin panel directly

Payout Card Setup

Add payout cards in Resource Management → Add Payout Card. Similar to collection cards, but these accounts will be used to send money.

10. Risk Control Settings

Each card can have independent risk control parameters:

SettingExampleEffect
Daily Transaction Limit10Auto-disable after 10 successful transactions today
Daily Amount Limit500000Auto-disable after 500,000 BDT received today
Balance Limit1000000Auto-disable when account balance reaches 1M BDT
Amount Range100-50000Only accept orders between 100-50,000 BDT
Fixed Amounts500,1000,2000Only accept orders of exactly 500, 1000, or 2000 BDT
When any limit is reached, the card automatically stops accepting new orders. You must manually re-enable it or adjust the limits.

11. Troubleshooting & FAQ

Q: OTP monitoring is not capturing messages

  • Check that all permission indicators are green in the monitoring app
  • Verify the server URL is correct (include https://, no trailing /)
  • Ensure the phone has a stable internet connection
  • Restart the monitoring app

Q: Order is stuck in "Processing" status

  • Check if the customer actually completed the payment
  • Verify the monitoring system is running and uploading transactions
  • If payment was made but not matched, use Manual Review to match the order

Q: Callback failed to merchant

  • Check merchant's server is reachable
  • Verify merchant's callback URL is correct
  • Use Manual Callback to resend the notification

Q: Card was auto-disabled

  • Check which risk control limit was triggered (daily count, amount, or balance)
  • Adjust the limits or manually re-enable the card

Q: How to add a proxy bKash card?

  • A "proxy card" is simply a bKash/Nagad account provided by a code merchant (third-party supplier)
  • Add it the same way as any other card — go to Resource Management → Add Card
  • Select the Code Merchant who provided the account
  • Enter the bKash/Nagad account number
  • Set up OTP monitoring on the code merchant's phone
  • The code merchant monitors transactions and manages the account
